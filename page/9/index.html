树莓派-Raspbian连接蓝牙设备

http://xlindo.com
所有文章遵循CC BY-NC-SA 4.0许可协议。请按许可使用。

测试环境

Raspberry B+ Raspbian 某USB蓝牙模块 thinkpad 小红点 蓝牙鼠标 其余正常配件

内容

安装必要的组件

apt-get install bluetooth bluez-utils bluez-compat blueman

该语句可能会要求权限，可执行

#sudo apt-get install bluetooth bluez-utils bluez-compat blueman

安装完后，插上USB蓝牙适配器，运行

#lsusb

看有没有正确识别Bluetooth Dongle设备,例如:

Bus 001 Device 005: ID 0a12:0001 Cambridge Silicon Radio, Ltd Bluetooth Dongle (HCI mode)

然后运行

# service bluetooth status

看看Bluetooth服务是否正常运行，正常的话应该是

[ ok ] bluetooth is running.

博主此时重启了一下，感觉RPi遇到问题重启，有可能就神奇的解决了

连接阶段

用这个命令查看周围可见设备的地址和名称：

# hcitool scan

显示

aa:bb:cc:dd:ee:ff Thinkpad Bluetooth Laser Mouse

运行 （后面换成你设备的地址）：

#bluez-simple-agent hci0 aa:bb:cc:dd:ee:ff

在这边输入一个密码（0000），回车。 如果失败则会出现Creating device failed: ……之类的。请重试。

务必确认蓝牙设备是在请求配对的状态。

将这个新设备加入到信任列表：

#bluez-test-device trusted aa:bb:cc:dd:ee:ff yes

用这个命令手工开始连接：

#bluez-test-input connect aa:bb:cc:dd:ee:ff

有时可能蓝牙鼠标会自己断开，执行上面这条命令重连一下就OK了

大功告成!

DnA - AVL树

http://xlindo.com
所有文章遵循CC BY-NC-SA 4.0许可协议。请按许可使用。

简介

在计算机科学中，AVL树是最先发明的自平衡二叉查找树。AVL树得名于它的发明者 G.M. Adelson-Velsky 和 E.M. Landis，他们在 1962 年的论文 “”An algorithm for the organization of information”” 中发表了它。

难点

AVL树的基本操作一般涉及运做同在不平衡的二叉查找树所运做的同样的算法。但是要进行预先或随后做一次或多次所谓的””AVL 旋转””。

旋转类型

1.LL型
2.LR型
3.RL型
4.RR型

算法上，将1和4归纳到一类，其中1采用右旋转，4采用左旋转。2和3归到另一类，2采用先左再右的双旋，3采用先右再左的双旋。

树莓派-CSI摄像头实时采集

http://xlindo.com
所有文章遵循CC BY-NC-SA 4.0许可协议。请按许可使用。

备注：在Windows上使用工具对树莓派进行远程的视频实时采集，为青年基金的工业相机控制组件项目铺路。

前言

对小车进行控制，不能只是单纯的遥控，这种遥控应该建立在能有视觉的基础上。当然，这种视觉只是一种初步的尝试，只是简单的一种获取。
本文是在Windows上通过mplayer及netcat对linux设备上的视频数据流进行获取，从而在Windows上能看到实时的视频。

硬件准备

树莓派小车（详见前阶段）
CSI摄像头

进行

  1. 系统启用相机
    sudo raspi-config
    将 “camera” 选择为 “enable”

    可以在终端中使用raspistill -o image.jpg获取一张图片或者使用raspivid -o video.h264 -t 10000获取一段10s的视频对相机进行简单测试。这里曾使用树莓派，在48小时左右的时间里每隔2s拍一张照片，对稳定性进行初步测试并且没有发现问题。

  2. 客户端为Linux系统：
    在终端中安装mplayer和netcat
    sudo apt-get install mplayer netcat
    将ip与树莓派设到同一网段，然后运行命令
    raspivid -t 999999 -o – | nc [insert the IP address of the client] 5001
    客户端为Windows系统:
    Install and run Linux instead.
    下载MPlayer和下载Netcat.
    将ip与树莓派设到同一网段，然后运行命令
    [Path to nc.exe]\nc.exe -L -p 5001 | [Path to mplayer.exe]\mplayer.exe -fps 31 -cache 1024 -

    netcat所做的就是在两台电脑之间建立链接并返回两个数据流，在这之后所能做的事就看你的想像力了。你能建立一个服务器，传输文件，与朋友聊天，传输流媒体或者用它作为其它协议的独立客户端。

  3. 通过PuTTY连接树莓派，使用raspivid发射视频数据流，此时便可以在客户端获得采集到的视频图像
    raspivid -t 999999 -o – | nc [insert the IP address of the client] 5001

  4. 使用注意：
    根据不同性能的电脑，mplayer 使用的参数作不同的设置，主要为fps和cache项.

总结

将netcat和mplayer相结合使用，可以在客户端对小车进行实时的远程视频采集，可以更好的掌控小车的行驶状态。

树莓派-树莓派小车跑起来

http://xlindo.com
所有文章遵循CC BY-NC-SA 4.0许可协议。请按许可使用。

对搭好的小车进行控制，第一步先使用无线网络，通过使用按键进行操作。对于良好控制的使用目标，将在下一阶段中增加对DSI接口摄像头的实时采集。

硬件准备

树莓派
无线模块
L298N电机驱动板
移动电源
搭好的小车
电池组
线材若干

进行

  1. 将小车搭好，将电机线连至L298N电机驱动板，使用电池组对驱动板进行供电，将供电线正极连至12V(12V~46V较佳)输入，负极连至接地端并将其与树莓派板共地。
  2. 使用移动电源对树莓派单独供电，使用无线模块将其连入局域网，使用其他电脑对其远程控制。这里使用PuTTY的SSH远程登陆。
  3. 将L298N驱动板上的IN1~IN4使用杜邦线与树莓派GPIO口连接，这里使用GPIO口的0~3。
    (树莓派GPIO请自行搜索)

我们使用的马达连接驱动板后,IN1 IN2 是控制底盘左边的马达, IN3 IN4是控制底盘右边的马达. 控制的原理拿左边的电机来讲: 依次把驱动板的IN1~IN2置为高电平，就可以驱动马达，也就是说，要把树莓派的2个GPIO输出口依次置为高电平。例如，假设用0代表低电平，1代表高电平的话，GPIO 0、1口的电平第一次被置为1、0，第二次被置为0、1，持续下去,电机就会运作。

代码

  1. 将代码写入car1.py

  2. 安装python的GPIO库:
    安装python编译环境
    sudo apt-get install python-dev
    安装GPIO库
    mkdir gpio
    cd gpio
    wget https://pypi.python.org/packages/source/R/RPi.GPIO/RPi.GPIO-0.5.7.tar.gz
    #或者到这里下载最新版本：https://pypi.python.org/pypi/RPi.GPIO
    tar xvzf RPi.GPIO-*.tar.gz
    cd RPi.GPIO-*/
    sudo python setup.py install

  3. 安装curses库:
    sudo apt-get install libncurses5-dev
    (如果发现找不到这个包，使用命令 sudo apt-get update 更新下包源)

  4. 运行：
    python car1.py
    此时可对小车运动进行控制，可以使用”w”、”a”、”d”、”x”进行方向控制，”s”终止程序。
  5. 源代码：

    总结

  • 电机需要在较高电压下才能实现转动速度的提升
  • 使用curses可以实现简单的交互

树莓派 - wiringPi GPIO 的使用

http://xlindo.com
所有文章遵循CC BY-NC-SA 4.0许可协议。请按许可使用。

前言

展开了对树莓派的学习之后，发现几乎所有的元器件都要通过树莓派上的接口连接，从而实现对其的控制。在前期的电机控制测试中用到了几个GPIO端口，在程序的编写中，为了实现控制，使用了wiringPi这个由Arduino衍生来的库。于是，在网上搜罗了一下该库的介绍。

这里，对于详细的电气方面的原理由于难以考证，在评论中发现参考的文章也可能有错误，所以，对于具体寄存器地址不作过多解释，只表明大意，重点在wiringPi代码内部的实现函数的讲解。

原理

树莓派平台的GPIO驱动，例如RPi.GPIO和WiringPi均采用直接操作GPIO寄存器的方式，树莓派的CPU采用博通的BCM2835，想要更好的了解树莓派的GPIO驱动实现就必须阅读BCM2835的数据手册。在BCM2835数据手册中需要认真关注两个内容：

  • 外设寄存器物理地址和外设虚拟地址的映射关系。在linux操作系统中，借助ARM内部的MMU，CPU外设物理地址映射成了虚拟地址，相应的物理地址都存在固定的关系。
  • GPFSELx、GPSETx、GPCLRx和GPLEVn寄存器。简单来说，GPFSELx为IO口方向或复用寄存器，负责IO口方向例如输入或输出；GPSETx为IO口输出寄存器，负责IO口输出逻辑高电平；GPCLRx寄存器同为IO口输出寄存器，不过和GPSETx相反，负责输出逻辑低电平。GPLEVx为IO口输入寄存器，负责IO口输入状态。

代码详解

代码可分为4个部分——wiringPiSetup初始化、pinMode设置IO为输出方向、digitalWrite输出高电平或低电平、delay系统延时函数。

总结

wiringPi可通过两种方式实现GPIO的驱动——第一，在用户空间直接操作寄存器（RAM），在用户空间操作寄存器（RAM）需要借助 /dev/mem；第二，利用/sys/class/gpio，通过操作文件的方式控制GPIO。在wiringPi中pin编号小于64为板载设备，例如GPIO0到GPIO7为板载设备，pin编号大于64为扩展设备，例如扩展的AD和DA芯片。最后可以使用nanosleep实现定时休眠。
