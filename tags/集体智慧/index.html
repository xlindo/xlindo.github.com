3 Discovering Groups

第二章可以使我们能找到与自己在某方面品味相似的人。拓展此思想，引入数据聚类data clustering的概念。本章将学到以下内容：

  • 从不同来源中构造算法所需数据；
  • 两种聚类算法；
  • 更多有关距离度量distance metrics的知识；
  • 简单的图形可视化代码；
  • 如何将异常复杂的数据集投影到二维空间。

聚类常被用于数据量很大data-intensive的应用中。如消费者购买行为；计量生物学computational biology中寻找相似行为的基因组等。

本书讨论集体智慧，故例子所涉及的都是多人贡献的不同信息的问题：

  1. 考察博客用户所讨论的话题，以及实用的特殊词汇，借此说明，可以由此产生分组方式；
  2. 考察社区网站，人们可以在这类网站上列举拥有和希望拥有的物品，我们将利用这些信息来示范，如何对人的意愿进行分组。

3.1 Supervised versus Unsupervised Learning 监督学习和无监督学习

利用样本输入和期望输出来学习如何预测的技术被称为监督学习supervised learning，如：

  • 神经网络 neural networks,
  • 决策树 decision trees,
  • 支持向量机 support-vector machines,
  • 贝叶斯过滤器 Bayesian filtering.

聚类是无监督学习unsupervised learning的一个例子，它不利用带有正确答案的样本数据进行训练training。其目的是在数据中找到某种结构，而答案并非数据本身。比如，聚类的结果不会说某位顾客会买什么，而是预测其适合某种时尚。其他非监督学习算法还有：

  • 非负矩阵因式分解 non-negative matrix factorization,
  • 自组织映射 self-organizing maps.

3.3 Hierarchical Clustering 分级聚类

分级聚类通过连续不断合并最为相似的群组，来构造一个层级结构。其中每个群组都是从单一元素开始的，在本章的例子中，这个单一元素就是博客。

在每次迭代过程中，分级聚类算法会计算每两个群组间的距离，并合并距离最近的两个群组。该过程重复进行，直到只剩一个。

可以使用树状图dendrogram来展现结果，这种绘制方式可以确定元素间的相似程度，并以此指示聚类的紧密程度。

2.7 Item-Based Filtering 基于物品的过滤

前文中基于用户的协作型过滤user-based collaborative filtering，要求我们使用来自每一位用户的全部评分，显然不适用于大用户量的场景。

基于物品的协作型过滤item-based collaborative filtering可以在拥有大量数据集的情况下，得出更好的结论，而且它允许大量任务预先执行，甚至独立于主应用之外单独进行计算。

其总体思路就是为每件物品预先计算好最为相近的其他物品：提供推荐前，查看目标用户的评价历史，选出靠前者，再构造出一个加权列表，其中包含了与这些选中物品最为相近的其他物品。其最为显著的特点是，物品间的比较不会像用户间的比较那样频繁变化。
