2 Persistent excitation

a signal $u(t)$ is persistently exciting of order $n$ if

  • the following limit exists:
  • the following matrix is positive definite

2.1 Examining the order of persistent excitation

2.1.1 white noise input of zero mean and variance $\lambda^{2}$

thus, white noise is persistently exciting of all orders

2.1.2 step input of magnitude $\lambda$

so, a step function is persistently exciting of order 1

2.1.3 impulse input: u(t) = 1 for t = 0 and 0 otherwise

so, an impulse is not persistently exciting of any order

2.2 Some results

  • An input signal is persistently exciting of order $2n$ can be used to consistently estimate parameters of a model of order $\leq n$.
  • A step function that is persistently exciting of order $1$, but not of greater order.
  • A PRBS with period $M$ is persistently exciting of order $M$, and can generate a $2^M − 1$ full length sequence.
  • An ARMA process is persistently exciting of any finite order.
  • A sum of $M$ sinusoids is persistently exciting of order $n=2M$ (if $\omega_{m} \neq 0,-\pi, \pi$)
  • A parametric model becomes more accurate in the frequency region where the input signal has a major part of its energy.

Summary

  • The choice of input signals determines the quality of the estimate.
  • The estimated model is more accurate in frequency regions where the input signal contains much energy.
  • An input signal has to be ’rich’ enough to excite all interesting modes of the system (PE of sufficiently high order).
  • the choice of input is imposed by the type of identification method
  • some often used signals include PRBS and ARMA processes

The choice of a signal input is important on the PE order, that it is larger than the number of parameters. —xlindo

Segmentation Index

  • A preliminary segmentation index can be written as
    • $\eta_{S E G}=\left|\Delta \hat{H}-\Delta H_{t}\right|$
    • where $\Delta \hat{H}$ is the differential entropy of the current model and $\Delta H_t$ is the differential entropy of the current point.

    • $\Delta H_{r e s}=\left|\Delta \hat{H}-\Delta H_{d a t a}(t)\right|$

Assume that the standard deviation of $\hat{y}_{t}$ is given as $\sigma_{\mathcal{y}}$ and the autocovariance at lag $k$ is given as $\gamma(k)$. Effects of Poles and Zeros

/wiki/Control_Systems/Poles_and_Zeros

As s approaches a zero, the numerator of the transfer function (and therefore the transfer function itself) approaches the value 0. When s approaches a pole, the denominator of the transfer function approaches zero, and the value of the transfer function approaches infinity. An output value of infinity should raise an alarm bell for people who are familiar with BIBO stability. We will discuss this later.

As we have seen above, the locations of the poles, and the values of the real and imaginary parts of the pole determine the response of the system. Real parts correspond to exponentials, and imaginary parts correspond to sinusoidal values. Addition of poles to the transfer function has the effect of pulling the root locus to the right, making the system less stable. Addition of zeros to the transfer function has the effect of pulling the root locus to the left, making the system more stable.

/wiki/QR_decomposition

In linear algebra, a QR decomposition (also called a QR factorization) of a matrix is a decomposition of a matrix A into a product A = QR of an orthogonal matrix Q and an upper triangular matrix R. Lennart Ljung, 1997

1 Introduction

  • The process of going from observed data to a mathematical model is fundamental in science and engineering.
  • dynamical models (difference or differential equations)
  • System Identification covers the problem of building models of systems where both when insignificant prior information is available and when the system’s properties are known, up to a few parameters (physical constants). Accordingly, one talks about black box and gray box models.
  • black box models there are familiar
    • linear models such as ARX and ARMAX,
    • and among non-linear black box models we have, e.g., Artificial Neural Networks (ANN).

1.1 The Problem

  • The area of system identification begins and ends with real data. Data are required to build and to validate models.
  • So, the bottom line of these examples is that we have collected input-output data from a process or a plant, and we need to extract information from these to find out (something about) the process’s dynamical properties.

1.2 Background and Literature System

  • System Identification has its roots in standard statistical techniques and many of the basic routines have direct interpretations as well known statistical methods such as Least Squares and Maximum Likelihood.
  • Maximum likelihood estimation was applied to difference equations (ARMAX models) [1965] and thereafter a wide range of estimation techniques and model parameterizations flourished.
  • The literature on System Identification is extensive.
    • except approaches in this book:
      • estimation of models from given frequency response measurement [Schoukens and Pintelon, 1991]
      • on-line model estimation [Ljung and Söderström, 1983]
      • non-parametric frequency domain methods [Brillinger, 1981]
    • IFAC series of Symposia on System Identification

1.3 Outline

  • The system identification procedure is characterized by four basic ingredients:
    • The observed data
    • A set of candidate models
    • A criterion of fit
    • Validation
  • The problem can be expressed as finding that model in the candidate set, that best describes the data, according to the criterion, and then evaluate and validate that model’s properties.
  • In general terms, a model will be a predictor of the next output $y(t)$ from the process, given past observations $Z^{t-1}$ and parameterized in terms of a finite-dimensional parameter vector, $\hat{y}(t | \theta)=g\left(\theta, Z^{t-1}\right)$

2.2.2 Second-Order Systems Unit-Step Response

Mass-Damping-Spring
  • $\ddot x+\frac{B}{m}\dot x+\frac{K}{m}x=F$
    • $\omega_n=\sqrt{\frac{K}{m}}$, Natural Frequency
    • $\zeta = \frac{B}{2\sqrt{Km}}$, Damping Ratio
1. Characteristic Equation
  • For response to initial condition: $F=0, x(0)=x_0, \dot x(0)=\dot x_0$
  • $\ddot x+2\zeta\omega_n\dot x+\omega_n^2x=0$
  • with $x(t)=e^{\lambda t}, \dot x(t)=\lambda e^{\lambda t}, \ddot x(t)=\lambda^2e^{\lambda t}$, obtain characteristic equation and solve and get characteristic roots $\lambda_{1,2}$ with $\zeta$ in the expression
  • Situations
    • Overdamped,$\zeta >1$
    • Critically damped, $\zeta =1$
    • Underdamped, $0<\zeta <1$
    • Undamped, $\zeta =0$
2. Tranfer Function
  • For Unit-Step Input, $U(s)=\frac{1}{s}$
    • $X(s)=U(s)G(s)=\frac{1}{s}G(s)$
    • $x(t)=\mathcal L^{-1}[X(s)]=1-e^{-\zeta\omega_n t}\sqrt{\frac{1}{1-\zeta^2}}sin(\omega_d t+\phi)$
    • Focus on underdamped

2.2.3 Assessment

  • $T_d$: Delay time, the time for rise to 50%
  • $T_r$: Rise time, the time for the first arrival to the final value
  • $M_p^\%$, Maximum percent overshoot, the vertical distance between the maximum peak
  • $T_{ss}$, settling time, the time required for the response curve to reach and stay within a certain strip(2%~$4\tau$, 5%~$3\tau$) about the final value is referred to as the settling time.
  • $T_p$, peak time

2.2.4 Frequency Response

  • $G(j\omega)$
  • Magnitude Response
    • $\frac{M_o}{M_i}=M$
  • Phase Response
    • $\phi_o - \phi_i = \phi$
  • Sinusoidal signal: invariable frequency
    • $u(t)=Asin{\omega t} + Bcos{\omega t} \sim M_i sin(\omega t+\phi_i)$
  • Frequency response actually is steady state response:
    • $x_{ss}(t) = K_1 e^{-j\omega t}+K_2 e^{j\omega t}$
    • $K_1, K_2$ are conjugate
  • $M_G=|G(j\omega)|$, if in $20log|G(j\omega)|\sim \omega\,\Rightarrow$Bode Plot
  • $\phi_G=\angle G(j\omega)$
  • Cut-Off Frequency
    • For Low-Pass Filter, $G(s)=\frac{a}{s+a}, s=j\omega$:
      • $|G(j\omega)|=\sqrt{\frac{1}{1+(\frac{\omega}{a})^2}}$
      • Cut-Off Freaquency: $\omega = a, |G(j\omega)| = 0.707$
    • For High-Pass Filter, $G(s)=\frac{s}{s+a}, s=j\omega$:
      • $|G(j\omega)|=\sqrt{\frac{1}{1+(\frac{a}{\omega})^2}}$

1 structure

The structure of a scientific Paper, Report, or Thesis

  • Abstract: Summarises
  • Introduction: Motivation & Objectives
  • Theory: theoretical underpinning or new theoretical perspective
  • Experimental / Simulation procedure : how it is tested or validated
  • Result & Discussion : outcomes of Experiments/Simulation → theory
  • Conclusions: Summarize the paper & clearly state overall results
  • References
  • Acknowledgements. Appendices

1.1 Abstract: Summarises

  • Background Motivation + Objectives:
    • 1~2 S
  • Theoretical Underpinning:
    • 1~2 S
  • Experimental Explanation
    • 0.5 S (in control) ~ 2+ (certain fields)
  • Results & Discussion
    • at least 0.5 S per objective (忌废话)
  • Conclusions/Future work
    • ~1 S

1.2 Introduction: Motivation & Objectives

  • Background/General Motivation
    • 1 P briefly states:
      • Why the field of research is important
      • give some bgs
  • Literature Review:
    • 2~3 P
      • what has been done
      • deficiencies or holes
        • analysed & discussed together
        • use transition words to show the relationship
        • 2 Styles:
          • Indirect: (present tense) emphasise results
          • Direct: (past tense) emphasize author
  • objectives:
    • 1 P states the point
      • 1~4 objectives
      • signposts & efficiencies in the current advanced

1.3 Theory: theoretical basis

  • Background:
    • explains key notation, concepts & results required to understand work
    • used to speed up on readers’ knowledge
    • range from a single figure & a few lines of text to a long sections with many subsections
  • relevant theory:
    • important or new info
    • a section per theoretical objective
    • (carefully) write to bring the reader to conclusion
    • Theorem’s proof:
      • State assumptions: limitations
      • State the fundamentals: previously accepted
      • combine the 2 above → new conclusion
      • State the conclusion

1.4 Procedure: how experiments/simulation was performed

  • sequentially arrange, so that it can be reproduced
  • a schematic is helpful to explain what is going on.

1.5 Results & Discussion: answer the objs or questions raised in the paper.

  • fundamental unit is some result: equation, formula, figure, or table
    • 1 P per unit
    • Structure:
      • Introduction: obj of interest .
        • 1 S
      • Description: how it was created
        • 1+ S
      • Result
        • 1 S
      • Discuss: Meaning
        • 1+S
        • Multi results can be combined into a single discussion
      • Conclusion

        1.6 Conclusions: summary with detailed results

  • Overall Summary: including motivation & methods
  • Summary of the objs: quantitative summary for each obj with actual numbers & figures
  • Extension: future or implication

1.7 Acknowledgements : Thanks.

  • for all authors
  • Separate

1.8 References

3. Logistical Regression:

  • classification : $y=0\ or\ 1$; $0 <= h_\theta(x) <= 1$
  • sigmoid function: $h_\theta(x) =\theta^Tx$ -> $g(\theta^Tx) = g(z) = \frac{1}{1+e^{-z}}$
  • LinearRegressiong cost function:
    • $ J(\theta) = \frac{1}{m}\sum_{i=1}^{m}\frac{1}{2}(h_\theta(x^{(i)})- y^{(i)})^2)$
  • Cost Function:
    • $cost(h_\theta(x),y) = -y\log(h_\theta(x))-(1-y)\log(1-h_\theta(x))$
    • $J(\theta)=-\frac{1}{m}\sum_{i=1}^{m}[y^{(i)}\log(h_\theta(x))+(1-y^{(i)})\log(1-h_\theta(x))]$
  • Gradien Descent
    • $\theta_j := \theta_j - \alpha\frac{\partial}{\partial\theta}J(\theta)$
    • Repeat{ $\theta_j := \theta_j - \alpha\sum(h_\theta(x^{(i)})-y^{(i)})x^{(j)}_j$ }
    • Vectorized implementation
  • Process:
    • Given $\theta$, compute:
      • $J(\theta)$
      • $\frac{\partial}{\partial\theta_j}J(\theta)$
      • Regularization(except i=0)
        • $J(\theta)=\frac{1}{2m}[\sum_{i=1}^{m}(h_\theta(x^{(i)})-y^{(i)})^2+\lambda\sum_{j=1}^{n}\theta_j^2]$
        • $\theta_j := \theta_j - \alpha[\frac{1}{m}\sum_{i=1}^{m}(h_\theta(x^{(i)})-y^{(i)})x^{(i)}_j+\frac{\lambda}{m}\theta_j]$
  • Optimization algorithms
    • Gradient descent
    • Conjugate gradient
    • BFGS
    • L-BFGS
  • Pros
    • Not need to manually pick $\alpha$
    • Often faster than gradient descent
  • Cons
    • More complex
  • Octave
    • fminunc:function minimization unconstrained

4. Neural Networks

  • Notations
    • $a_i^{(j)}$: “activation” of unit i in layer j
    • $\theta^{(j)}$: matrix of weights controlling function mapping from layer j to layer j+1
      • e.g. $a_1^{(2)}=g(\theta_{10}^{(1)}x_0+\theta_{11}^{(1)}x_1+\theta_{12}^{(1)}x_2+\theta_{13}^{(1)}x_3)$
      • $h_\theta(x) = a_1^{(3)}$
      • $z^{(2)}=\Theta^{(1)}x=\Theta^{(1)}a^{(1)}$

5. NN BP

  • Logistic Regression: $J(\theta) = -\frac{1}{m}\sum_{i=1}^{m}[y^{(i)}\log((h_\theta(x^{(i)}))+(1-y^{(i)})\log(1-h_\theta(x^{(i)}))]+\frac{\lambda}{2m}\sum_{j=1}^{n}\theta^2_j$
  • Denote $h_\theta(x)_k$as $k^{th}$ -> NN: $J(\Theta)$
    • L: sum of layers; $s_l$:no. of units; k: no. of output units
    • Logistic Regression: $J(\Theta) = -\frac{1}{m}\sum_{i=1}^{m}\sum_{k=1}^{K}[y_k^{(i)}\log((h_\theta(x^{(i)})_k)+(1-y_k^{(i)})\log(1-(h_\theta(x^{(i)})_k))] + \frac{\lambda}{2m}\sum_{l=1}^{L-1}\sum_{i=1}^{s_l}\sum_{j=1}^{s_{l+1}}(\Theta_{j,i}^{(l)})^2$
  • Forward propagation
    • $a^{(1)}=x$
    • $z^{(2)}=\Theta^{(1)}a^{(1)}$
    • $a^{(2)}=g(z^{(2)})$; (add $a_0^{(2)}$)
    • $z^{(3)}=\Theta^{(2)}a^{(2)}$
    • $a^{(3)}=g(z^{(3)})$;(add $a_0^{(3)}$)
    • $z^{(4)}=\Theta^{(3)}a^{(3)}$
    • $a^{(4)}=g(z^{(4)})$
  • Backpropagration
    • intuition: $\delta_j^{(l)}$ = “error” of node j in layer l
    • PROCESS
        1. Given data set
        1. set$\Delta_{ij}^{(l)}:=0$ for all i,j
        1. for t=1 to m:
          • (1) set $a^{(1)}=x$
          • (2) FP-> compute $a^{(l)}$ for l=2,3,…,L
          • (3) compute $\delta^{(L)}=a^{(L)}-y^{(t)}$
          • (4) compute $\delta^{(L-1)},\delta^{(L-2)},\dots,\delta^{(2)}$, using $\delta^{(l)}=((\Theta^{(l)})^T\delta^{(l+1)}.a^{(l)}.(1-a^{(l)})$
          • (5) $\Delta_{ij}^{(l)}:=\Delta_{ij}^{(l)} +a_{j}^{(l)}\delta_{i}^{(l)}$
          • e.g. $\delta^{(3)}=(\Theta^{(3)})^T\delta^{(4)}.*g^{‘}(z^{(3)})$
          • e.g $g^{‘}(z^{(3)}) = a^{(3)}.*(1-a^{(3)})$
          • $\delta^{(4)}\rightarrow\delta^{(3)}\rightarrow\delta^{(2)}\rightarrow\delta^{(1)}$
        1. if $j\neq0$, $D_{i,j}^{(l)}:=\frac{1}{m}(\Delta_{ij}^{(l)} + \lambda\Theta_{ij}^{(l)})$; if $j=0$, $D_{i,j}^{(l)}:=\frac{1}{m}\Delta_{ij}^{(l)}$
        1. $\frac{\partial}{\partial\Theta_{i,j}^{(l)}}J(\Theta)=D_{i,j}^{(l)}$
  • Gradient checking
    • $\frac{\partial}{\partial\theta}J(\theta)\approx\frac{J(\theta+\epsilon)-J(\theta-\epsilon)}{2\epsilon}; \epsilon=10^{-4}$
    • delete these checking code when implement
  • Train a NN
      1. randomly initialize weights, cannot be set as 0
      1. FP to get $h_\theta(x^{(i)})$
      1. compute $J(\Theta)$
      1. compute $\frac{\partial}{\partial\Theta_{i,j}^{(l)}}J(\Theta)$
      1. use gradient check to compare BP vs. numerical estimate
