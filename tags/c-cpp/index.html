三段程序说明“类”相关的public、private、protected的概念

xlindo 2017年12月14日 原文：深入理解C++中public、protected及private用法

一般性概念

  1. 类的一个特征就是封装，public和private作用就是实现这一目的。所以：
    用户代码（类外）可以访问public成员而不能访问private成员；private成员只能由类成员（类内）和友元访问。

  2. 类的另一个特征就是继承，protected的作用就是实现这一目的。所以：
    protected成员可以被派生类对象访问，不能被用户代码（类外）访问。

三种继承

另外，有public, protected, private三种继承方式，它们相应地改变了基类成员的访问属性：

  1. public继承：基类public成员，protected成员，private成员的访问属性在派生类中分别变成：public, protected, private
  2. protected继承：基类public成员，protected成员，private成员的访问属性在派生类中分别变成：protected, protected, private
  3. private继承：基类public成员，protected成员，private成员的访问属性在派生类中分别变成：private, private, private

0 概述

C++标准库提供了更为安全、更为灵活的数据集处理方式。STL最初由hp实验室开发，最主要的两个特点是：

  • 数据结构与算法的分离
  • 非面向对象本质

2 六大组件

  • container：如list、vector、deques，以模板类的方法提供
  • iterator：访问容器中对象的方法
  • algorithm：用来操作容器类中的模板函数
  • function object：仿函数functor，其实就是重载了（）操作符的struct
  • adaptor：迭代适配器
  • allocator：对象的创建与销毁

2.1 container

  • sequence container：vector,deque,list:
    • vector：可以随机存取元素，但是中部或头部安插元素比较费时
    • deque：可以随机存取元素，在中部安插元素比较费时
    • list：不提供随机存取，任何位置都比较迅速
  • associated container：set,multiset,map,mulitimap:
    • set/multiset：内部的元素依据其值自动排序，set相同数值的元素只能出现一次，multiset内可包含多个数值相同的元素，内部使用红黑树实现，查找速度快
    • map/multimap：键值对，内部元素依据值自动排序，map中相同数值的元素只能出现一次，而multimap可包含多个数值相同的元素，内部使用红黑树实现，查找速度快

4 迭代器技术

4.1 流

对于迭代器，有另一种方法使用流和标准函数。理解的要点是将输入/输出流作为容器看待。因此，任何接受迭代器参数的算法都可以和流一起工作。copy(v.begin(), v.end(), ostream_iterator<int>(cout, "\t"));这句话有点神奇。

4.2 插入

如通过使用front_inserter迭代器对象，可以只用单个copy()语句就完成将矢量中的对象插入到链表前端的操作：copy(dVector.begin(), dVector.end(), front_inserter(dList));。一般来说，插入到链表这样的与位置无关的容器结构更为有效。

三种插入迭代器如下：

  • 普通插入器 将对象插入到容器任何对象的前面。
  • Front inserters 将对象插入到数据集的前面——例如，链表表头。
  • Back inserters 将对象插入到集合的尾部——例如，矢量的尾部，导致矢量容器扩展。

5.5 绑定器函数对象

一个绑定器使用另一个函数对象f()和参数值V创建一个函数对象。被绑定函数对象必须为双目函数，也就是说有两个参数，A和B。STL中的绑定器有：

  • bind1st()创建一个函数对象，该函数对象将值V作为第一个参数A。
  • bind2nd()创建一个函数对象，该函数对象将值V作为第二个参数B。

Algorithm count_if()计算满足特定条件的元素的数目。 这是通过将一个函数对象和一个参数捆绑到为一个对象，并将该对象作为算法的第三个参数实现的。 注意这个表达式:

bind1st(greater<int>(), 8)
该表达式将greater<int>()和一个参数值8捆绑为一个函数对象。由于使用了bind1st()，所以该函数相当于计算下述表达式：

8 > q
表达式中的q是容器中的对象。因此，完整的表达式

count_if(aList.begin(), aList.end(), bind1st(greater<int>(), 8), k);
计算所有小于或等于8的对象的数目。

5.6 否定函数对象

所谓否定(negator)函数对象，就是它从另一个函数对象创建而来，如果原先的函数返回真，则否定函数对象返回假。有两个否定函数对象：not1()和not2()。not1()接受单目函数对象，not2()接受双目函数对象。否定函数对象通常和帮定器一起使用。例如，上节中用bind1nd来搜索q<=8的值：

count_if(aList.begin(), aList.end(), bind1st(greater<int>(), 8));

如果要搜索q>8的对象，则用bind2st。而现在可以这样写：

start = find_if(aList.begin(), aList.end(), not1(bind1nd(greater<int>(), 6)));
你必须使用not1，因为bind1nd返回单目函数。

前言

理解两个库，大致就一句话：stl是c++标准库的一个子集，c++的标准库主要有三部分：c标准库的c++版本；c++的IO库；c++的STL库。

  • C标准库的c++版本：原来的c版本是<*.h>:，c++版本是<c*>:
  • C++的IO库：最常用的是iostream，fstream
  • C++的stl库：下面会详细理解

c++标准库百度百科上给的解释：它是类库和函数的集合，其使用核心语言写成，由c++标准委员会制定，并不断的维护和更新的库，c++的标准库文件总共在50个标准头文件中定义，

标准库分为10类

1 标准库中与语言支持功能相关的头文件

  1. <cstddef>:定义宏NULL和Offsetof，以及其他标准类型的size_t和ptrdiff
  2. <limits>:提供与基本数据类型相关的定义。例如，对于每个数值数据类型，它定义了可以表示出来的最大值和最小值以及二进制数字的位数。
  3. <climits>:提供了与基本整数数据类型相关的C样式定义。这些信息的c++样式定义在<limits>中
  4. <cfloat>:提供与基本浮点型数据相关的C样式定义。这些c++的样式定义在<limits>:中
  5. <cstdlib>:提供支持程序启动和终止的宏和函数。这个头文件还声明了许多其他杂项函数，例如搜索和排序函数，从字符串转换成数值的函数， 他与对应的标准c头文件stdlib.H不同，定义了abort(VOID).abort函数还有额外的功能，它不为静态或者自动对象调用析构函数，也不调用atexit()函数，它还定义了exit()函数的额外功能，可以释放静态对象，以注册的逆序调用用atexit注册的函数。清楚并关闭所打开的c流，把控制权返回给主机环境
  6. <new>:支持动态内存分配
  7. <typeinfo>:支持变量在运行期间的类型标识
  8. <exception>:支持异常处理，这是处理程序中可能发生的错误的一种方式
  9. <cstdarg>:支持接收数量可变的参数的函数，记载调用函数时，可以给函数传送数量不等的数据项，他定义了宏va_arg,va_end,va_start,以及va_list类型
  10. <csetjmp>:为c样式的非本地跳跃提供函数。这些函数在c++中不常用
  11. <csignal>:为中断处理体从c样式支持

2 支持流输入输出的头文件

  1. <iostream>:支持标准流cin、cout、cerr和clog的输入和输出，并且支持多字节的输入输出
  2. <iomanip>:提供操纵程序，允许改变流的状态，从而改变输出的格式
  3. <ios>:定义iostream的基类
  4. <istream>:为管理输入流缓存区的输出定义模板类
  5. <ostream>:为管理输出流缓存区的输出定义模板类
  6. <sstream>:支持字符串的流输入输出
  7. <fstream>:支持文件流输入输出
  8. <iosfwd>:为输入输出对象提供向前的声明
  9. <streambuf>:支持流输入和输出的缓存
  10. <cstdio>:为标准流提供c样式的输入输出
  11. <cwchar>:支持多字节字符的输入输出

3 与诊断功能相关的头文件

  1. <stdexcept>:定义标准异常，异常时处理错误的方式
  2. <cassert>:定义断言宏，用于检查运行期间的情形
  3. <cerrno>:支持c样式的错误信息

4 定义工具函数的头文件

  1. <utility>:定义重载的关系运算符，简化关系运算符的写入，它还定义了pair类型
  2. <functional>:定义了许多函数对象类型和支持函数对象的功能，函数对象是支持operator()函数调用运算符的任意对象
  3. <memory>:给容器、管理内存的函数和auto_ptr模板类定义标准内存分配器
  4. <ctime>:支持系统时钟函数

5 支持字符串处理的头文件

  1. <string>:为字符串类型提供支持和定义
  2. <cctype>:单字节字符类别
  3. <cwctype>:多字节字符类别
  4. <cstring>:为处理非空字节序列和内存块提供函数，这不同于对用的标准的c库头文件， 几个c样式字符串的一般c库函数被返回值为const和非const的函数对替代了
  5. <cwchar>:为处理执行I/O和转换多字节序列提供函数，这不同于对应的标准c库头文件
  6. <cstdlib>:为把单字节字符串转换为数值. 在多字节字符和多字节字符串之间提供函数

6 定义容器类的模板的头文件

  1. <vertor>:定义vector序列模板，这是一个大小可以重新设置的数组类型，比普通的数组更安全，更灵活
  2. <list>:定义list序列模板，这是一个序列的链表，常常在任意位置插入和删除元素
  3. <deque>:定义deque序列模板，支持在开始和结尾的高效插入和删除操作
  4. <queue>:为队列（先进先出）数据结构定义序列适配器queue和priority_queue
  5. <stack>:为堆栈（后进先出）数据结构定义序列适配器stack
  6. <map>:map是一个关联容器类型，允许根据键值是唯一的，且按照升序存储。
  7. <set>:set是一个关联容器类型，用于以升序方式存储唯一值。
  8. <bitset>:为固定长度的位序列定义bitset模板，它可以看做固定长度紧凑型bool数组

7 支持迭代器的头文件

  1. <iterator>:给迭代器提供定义和支持

8 有关算法的头文件

  1. <algorithm>:提供一组给予算法的函数，包括置换、排序、合并和搜索
  2. <cstdlib>:声明C标准库函数bsearch()和qsort()，进行搜索和排序
  3. <ciso646>:允许在代码中使用and代替&&

9 有关数值操作的头文件

  1. <complex>:支持复杂数值的定义和操作
  2. <valarray>:支持数值矢量的操作
  3. <numeric>:在数值序列上定义一组一般数学操作
  4. <cmath>:这是c数学库，其中还附加了重载函数，以支持c++约定
  5. <cstdlib>:提供的函数可以提取整数的绝对值，对整数进行取余数操作

10 有关本地化的头文件

  1. <locale>:提供的本地化包含字符类别，排序序列以及货币和日期表示
  2. <clocale>:对本地化提供c样式支持

0 ANSI标准库

标准库不是C语言本身的一部分，其函数、类型以及宏分别在下面的头文件中定义：

以下划线开头的外部标识符保留给标准库使用，同时，以下划线、两个下划线或大写字母开头的标识符也保留给标准库使用。

1 输入输出

该头文件中定义的输入和输出函数、类型以及宏的数目几乎占整个标准库的三分之一。

1.1 文件操作

  • FILE *fopen(const char *filename, const char *mode)：其中mode可以是"r"、"w"、"a"、"r+"、"w+"、"a+"之一。
    后三种方式允许对同一文件进行读写。在读写交叉过程中，必须调用fflush或文件定位函数。一次最多打开FOPEN_MAX个文件。
  • FILE *freopen(const char *filename, const char * mode, FILE *stream)：该函数以mode方式打开指定文件，并将其关联到stream指定的流。常用于改变与stdin、stdout和stderr相关联的文件。
  • int fflush(FILE *stream)：对输出流，将写入缓冲区但尚未保存的数据保存；对输入，结果未定义。如果发生错误返回EOF，否则返回0。fflush(NULL)将清洗所有输出流。
  • int fclose(FILE *stream)：该函数将所有未写入的数据写入stream，丢弃缓冲区中所有未读入数据，并释放自动分配的缓冲区，关闭流。如果发生错误返回EOF，否则返回0。
  • int remove(const char *filename)：删除指定文件，若失败，返回一个非0值。
  • int rename(const char *oldname, const char *newname)：重命名，若失败，返回一个非0值。
  • FILE *tmpfile(void)：以"wb+"创建一个临时文件，该文件在被关闭或程序正常结束时被删除。创建成功，返回一个流；创建失败返回NULL。
  • char *tmpnam(char s[L_tmpnam])：tmpnam(NULL)创建一个与现有文件名不同的字符串，并返回一个指向一内部数组的指针。tmpnam(S)函数把创建的字符串保存到数组S中，并将它作为函数值返回。该函数只是用于创建一个名字，而不是一个文件。
  • int setvbuf(FILE *stream, char *buf, int mode, size_t size)：在执行读写或其他操作前必须执行此函数，该函数控制流的缓冲。当mode为_IOFBF时，进行完全缓冲；当mode为_IOLBF时，对文本文件进行缓冲；当mode为_IONBF时，不缓冲。如果buf的值不是NULL，则将buf指向的区域作为缓冲区，否则将分配一个缓冲区。size决定缓冲区长度。如果出错，返回一个非0值。
  • void setbuf(FILE *stream, char *buf)：如果buf值为NULL，关闭流的缓冲；否则等价于(void)setvbuf(stream, buf, _IOFBF， BUFSIZE)。

1.2 格式化输出

  • int fprintf(FILE *stream, const char *format, ...)：按照说明的格式对输出进行转换，并写到流中。返回实际写入的字符，否则返回负值。格式串由普通字符和转换说明组成。转换说明以%开头，以转换字符结束。其间可以依次包括：

    • 标志，用于修改转换说明：
      • -：指定被转换的参数在其字段内左对齐；
      • +：指定输出的数前加上正负号；
      • 空格：如果第一个字符不是正负号，则加一个空格；
      • 0：当输出长度小于字段宽度，前加0填充；
      • #：指定另一种输出形式：如果为o转换，则第一个数字为0，如果为x或X转换，则指定在输出的非0值前加0x或0X；对于e、E、f、g、G转换，指定输出总包含一个小数点；对于g或G，保留末尾的0。
    • 一个数值，指定最小字段宽度
    • 点号，用于分隔字段宽度和精度
    • 精度：对于字符串，它指定打印字符的最大个数；对于e、E或f转换，指定小数点后的数字位数；对于g或G，指定有效数字位数；对于整型，指定打印的数字位数

    • 长度修饰h(short)、l(long)或L(long double)

      宽度和精度都可以用*指定，则该值将通过转换下一个参数（必须为int）计算得到。

  • int printf(const char *format, ...)：等价于fprintf(stdout, ...)
  • int sprintf(char *s, const char *format, ...)：与printf基本相同，但输出将被写到s中，并以\0结束
  • vprintf、vfprintf和vsprintf这三个函数和对应的printf等价，但用宏va_start初始化的arg代替了可变参数

1.6 文件定位

  • int fseek(FILE *stream, long offset, int origin)：设置文件位置，后续操作从新位置开始。其中origin可以被设置为SEEK_SET（开始）、SEEK_CUR（当前）或SEEK_END（末尾）。对于文本流，offset必须设置为0或是ftell返回的值（此时origin必须是SEEK_SET)。该函数出错时返回一个非0值。
  • long ftell(FILE *stream)：返回stream当前位置，出错时返回-1L。
  • void rewind(FILE *stream)：等价于fseek(fp, 0L, SEEK_SET)、clearerr(fp)的执行结果。
  • int fgetpos(FILE *stream, fpos_t *ptr)：记录当前位置，供fsetpos使用。
  • int fsetpos(FILE *stream, const fpos_t *ptr)：设置位置。

1.7 错误处理

当发生错误或到达文件末尾时，标准库中许多函数都会设置状态指示符。另外，errno（<errno.h>）可以包含一个错误编号。

  • void clearerr(FILE *stream)：清除与流先关的文件结束符和错误指示符。
  • int feof(FILE *stream)：如果设置了与流相关的文件结束指示符，将返回一个非0值。
  • int ferror(FILE *stream)：如果设置了与流相关的错误指示符，将返回一个非0值。
  • void perror(const char *s)：将打印s以及与errno中整型值相应的错误信息，功能类似于fprintf(stderr, "%s: %s\n", s, "error message");。

7 可变参数表

该头文件提供了遍历未知数目和类型的函数参数表功能。假定函数f带有可变数目的实际参数，lastarg是它最后一个命名的形式参数。那么在函数内声明一个va_list的变量ap，它将依次指向每个实际参数：va_list ap;

在访问未命名的参数前，必须使用va_start宏初始化一次：va_start(va_list ap, lastarg);

此后，每次执行宏va_arg都将产生一个与下一个未命名的参数具有相同类型和数值的值，同时它还修改ap，以使得下一次执行va_arg时返回下一个参数：type va_arg(va_list ap, type);

在所有参数处理完毕之后，且在退出函数前，必须调用宏va_end一次：void va_end(va_list ap);

9 信号

该头文件提供了一些处理程序运行期间印发的各种异常条件的功能，比如来自外部的中断信号或程序执行错误引起的中断信号。

void (*signal(int sig, void (*handler)(int)))(int)：如果handler的值是SIG_DFL，则采用由实现定义的默认行为；若为SIG_IGN，则忽略；否则，调用handler指向的函数，有效的信号包括：

  • SIGABRT：异常终止
  • SIGFPE：算术运算出错，如除数为0或溢出
  • SIGILL：非常函数映像，如非法命令
  • SIGINT：用于交互式目的的信号，如中断
  • SIGSEGV：非法存储器访问，如访问不存在的内存单元
  • SIGTERM：发送给程序的终止请求

对于特定的信号，信号将返回handler的前一个值，如出现错误，返回SIG_ERR。

int raise(int sig)发送信号sig，若不成功，则返回一个非0值。

10 日期与时间

clock_t和time_t是两个表示时间的算术类型，struct tm用于保存日历时间的各个构成部分，包括：int tm_sec;、int tm_min;、int tm_hour;、int tm_mday;、int tm_mon;、int tm_year;、int tm_wday;、int tm_yday;、int tm_isdst;（夏令时标记）。

  • clock_t clock(void)：返回程序开始执行后占用的处理器时间，单位为秒。
  • time_t time(time_t *tp)：返回当前日历时间。
  • double difftime(time_t time2, time_t time1)：返回时间差，秒。
  • time_t mktime(struct tm *tp)：转换*tp的当地时间到日历时间。
  • char *asctime(const struct tm *tp)：格式化时间。
  • char *ctime(const time_t *tp)：转换日历时间到当地时间，等价于asctime(localtime(tp))。
  • struct tm *gmtime(const time_t *tp)：转换日历时间到UTC。
  • struct tm *localtime(const time_t *tp)：转换日历时间到当地时间。
  • size_t strftime(char *s, size_t smax, const char *fmt, const struct tm *tp)：格式化时间。
