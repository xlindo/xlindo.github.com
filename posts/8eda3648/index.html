预备知识

Redis优点

  1. 快：每秒110000次SET，每秒81000次GET；
  2. 支持丰富的数据类型：列表、集合、排列集和散列等；
  3. 操作具有原子性：确保多个客户端访问时，Redis接收最新的值；
  4. 多实用工具：缓存、消息队列、短期数据等。

与其他键值存储系统对比

  1. Redis是内存数据库，但在磁盘数据库上是持久的；
  2. Redis可以将数据复制到任意数量的从机。

安装配置

  • 安装redis-server，安装redis-cli，运行ping。
  • 远程连接redis-cli -h host -p port -a password。

键命令

编号 命令 描述
1 DEL key 删除指定键（如果存在）
2 DUMP key 返回存储在指定键的值的序列化版本
3 EXISTS key 检查键是否存在
4 EXPIRE key seconds 设置键在指定秒数后过期
5 PERSIST key 永久化键
6 PTTL key 获取键的剩余到期（毫秒）
7 TYPE key 返回存储在键中的值的数据类型

客户端连接

在配置的监听TCP端口和Unix套接字上等待和接受客户端的连接（如果已启用）。当接受新的客户端连接时，执行以下操作：

  • 由于Redis使用复用和非阻塞IO，因此客户端套接字处于非阻塞状态；
  • 设置TCP_NODELAY选项是为了确保连接不延迟；
  • 创建可读文件事件，以便Redis能够在套接字上读取新数据时收集客户端查询。

设置最大客户数

redis 127.0.0.1:6379> CONFIG get maxclients
设置
redis 127.0.0.1:6379> redis-server --maxclients 100000

2 CLIENT KILL 关闭指定的客户端连接

管道

Redis是一个TCP服务器，支持请求/响应协议，通过以下步骤完成：

  • 客户端向服务器发送请求，通常以阻塞的方式;
  • 服务器处理命令并将响应发送到客户端。

意义及示例

分区

分区是将数据拆分为多个Redis实例的过程，因此每个实例只包含一部分键。

  • 范围分区：将对象的范围映射到特定的Redis实例；
