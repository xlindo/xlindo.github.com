4.1 Overﬂow and Underﬂow

  • The fundamental difficulty in performing continuous math on a digital computer is that we need to represent infinitely many real numbers with a finite number of bit patterns.
    • we incur some approximation error
      • In many cases, this is just rounding error 舍入误差
        • can cause algorithms that work in theory to fail in practice
  • Underflow 下溢 occurs when numbers near zero are rounded to zero.
    • e.g. divided by zero
  • Overflow 上溢 occurs when numbers with large magnitude are approximated as $\infty$ or $-\infty$.
  • One example of a function that must be stabilized against underflow and overflow is the softmax function, which is often used to predict the probabilities associated with a multinoulli distribution.
    • $\operatorname{softmax}(\boldsymbol{x})_{i}=\frac{\exp \left(x_{i}\right)}{\sum_{j=1}^{n} \exp \left(x_{j}\right)}$
    • Both of these difficulties can be resolved by instead evaluating $\operatorname{softmax}(\boldsymbol{z})$ where $\boldsymbol{z}=\boldsymbol{x}-\max _{i} x_{i}$.
      • Subtracting $\max _{i} x_{i}$ results in the largest argument to exp being $0$, which rules out the possibility of overflow.
      • Likewise, at least one term in the denominator has a value of $1$, which rules out the possibility of underflow in the denominator leading to a division by zero.
  • Developers of low-level libraries should keep numerical issues in mind when implementing deep learning algorithms.

4.2 Poor Conditioning

  • Conditioning refers to how rapidly a function changes with respect to small changes in its inputs.
    • rounding errors in the inputs can result in large changes in the output.
  • Consider the function $f(\boldsymbol{x})=\boldsymbol{A}^{-1} \boldsymbol{x}$. When $\boldsymbol{A} \in \mathbb{R}^{n \times n}$ has an eigenvalue decomposition, its condition number is
    • $\max _{i, j}\left|\frac{\lambda_{i}}{\lambda_{j}}\right|$
    • When this number is large, matrix inversion is particularly sensitive to error in the input.

4.3 Gradient-Based Optimization

  • We usually phrase most optimization problems in terms of minimizing $f(x)$.
  • The function we want to minimize or maximize is called the objective function , or criterion . When we are minimizing it, we may also call it the cost function, loss function , or error function .
  • We often denote the value that minimizes or maximizes a function with a superscript $∗$.
    • $\boldsymbol{x}^{*}=\arg \min f(\boldsymbol{x})$
  • Suppose we have a function $y = f(x )$
    • derivative, $f^{\prime}(x)$ or as $\frac{d y}{d x}$ gives the slope of $ f(x )$ at the point $x$
      • it specifies how to scale a small change in the input to obtain the corresponding change in the output
      • $f(x+\epsilon) \approx f(x)+\epsilon f^{\prime}(x)$
    • gradient descent， We can thus reduce $f(x )$ by moving $x$ in small steps with the opposite sign of the derivative.
    • Points where $f^{\prime}(x)=0$ are known as critical points , or stationary points .
    • A local minimum is a point where $f(x )$ is lower than at all neighboring points, so it is no longer possible to decrease $f(x )$ by making infinitesimal steps.
      • vice versa, local maximum
      • or a saddle point, which has neighbors that are both higher and lower than the point itself.
    • A point that obtains the absolute lowest value of $f(x )$ is a global minimum.
    • The gradient generalizes the notion of derivative to the case where the derivative is with respect to a vector:
      • the gradient of $f$ is the vector containing all the partial derivatives, denoted $\nabla_{\boldsymbol{x}} f(\boldsymbol{x})$
        • Element $i$ of the gradient is the partial derivative of $f$ with respect to $x_i$.
      • The directional derivative in direction $\boldsymbol{u}$ (a unit vector) is the slope of the function $f$ in direction $\boldsymbol{u}$ .
        • $\frac{\partial}{\partial \alpha} f(\boldsymbol{x}+\alpha \boldsymbol{u}) = \boldsymbol{u}^{\top} \nabla_{\boldsymbol{x}} f(\boldsymbol{x})$, when $\alpha= 0$.
    • To minimize $f$ , we would like to find the direction in which $f$ decreases the fastest.
      • $\min _{\boldsymbol{u}, \boldsymbol{u}^{\top} \boldsymbol{u}=1} \boldsymbol{u}^{\top} \nabla_{\boldsymbol{x}} f(\boldsymbol{x}) = \min _{\boldsymbol{u}, \boldsymbol{u}^{\top} \boldsymbol{u}=1}|\boldsymbol{u}|_{2}\left|\nabla_{\boldsymbol{x}} f(\boldsymbol{x})\right|_{2} \cos \theta$
      • Substituting in $|\boldsymbol{u}|_{2}=1$ and ignoring factors that do not depend on $\boldsymbol{u}$, this simplifies to
        • $\min _{\boldsymbol{u}} \cos \theta$
        • This is minimized when $\boldsymbol{u}$ points in the opposite direction as the gradient.
          • In other words, the gradient points directly uphill, and the negative gradient points directly downhill.
      • This is known as the method of steepest descent, or gradient descent.
    • Steepest descent proposes a new point
      • $\boldsymbol{x}^{\prime}=\boldsymbol{x}-\epsilon \nabla_{\boldsymbol{x}} f(\boldsymbol{x})$
        • where $\epsilon$ is the learning rate
    • Steepest descent converges when every element of the gradient is zero (or, inpractice, very close to zero).
    • Ascending an objective function of discrete parameters is called hill climbing.

4.3.1 Beyond the Gradient: Jacobian and Hessian Matrices

  • Jacobian matrix： Sometimes we need to find all the partial derivatives of a function whose input and output are both vectors. The matrix containing all such partial derivatives is known as a Jacobian matrix.
    • Specifically, if we have a function $\boldsymbol{f} : \mathbb{R}^{m} \rightarrow \mathbb{R}^{n}$, then the Jacobian matrix $\boldsymbol{J} \in \mathbb{R}^{n\times m} $ of $\boldsymbol{f}$ is defined such that
      • $J_{i, j}=\frac{\partial}{\partial x_{j}} f(\boldsymbol{x})_{i}$
    • second derivative: a derivative of a derivative
      • $\frac{\partial^{2}}{\partial x_{i} \partial x_{j}} f$
      • In a single dimension
        • $\frac{d^{2}}{d x^{2}} f$ by $f^{\prime \prime}(x)$
      • We can think of the second derivative as measuring curvature .
        • With negative curvature, the cost function actually decreases faster than the gradient predicts.
        • With no curvature, the gradient predicts the decrease correctly.
        • With positive curvature, the function decreases more slowly than expected and eventually begins to increase, so steps that are too large can actually increase the function inadvertently.
  • Hessian matrix: When our function has multiple input dimensions, there are many second derivatives.
    • $\boldsymbol{H}(f)(\boldsymbol{x})_{i, j}=\frac{\partial^{2}}{\partial x_{i} \partial x_{j}} f(\boldsymbol{x})$
    • Equivalently, the Hessian is the Jacobian of the gradient.
    • Because the Hessian matrix is real and symmetric, we can decompose it into a set of real eigenvalues and an orthogonal basis of eigenvectors.
      • The maximum eigenvalue determines the maximum second derivative, and the minimum eigenvalue determines the minimum second derivative.
    • The (directional) second derivative tells us how well we can expect a gradient descent step to perform.
    • We can make a second-order Taylor series approximation to the function $f(\boldsymbol{x})$ aroungd the current point $\boldsymbol{x}^{(0)}$:
      • $f(\boldsymbol{x}) \approx f\left(\boldsymbol{x}^{(0)}\right)+\left(\boldsymbol{x}-\boldsymbol{x}^{(0)}\right)^{\top} \boldsymbol{g}+\frac{1}{2}\left(\boldsymbol{x}-\boldsymbol{x}^{(0)}\right)^{\top} \boldsymbol{H}\left(\boldsymbol{x}-\boldsymbol{x}^{(0)}\right)$
      • with learning rate $\epsilon$
      • $f\left(\boldsymbol{x}^{(0)}-\epsilon \boldsymbol{g}\right) \approx f\left(\boldsymbol{x}^{(0)}\right)-\epsilon \boldsymbol{g}^{\top} \boldsymbol{g}+\frac{1}{2} \epsilon^{2} \boldsymbol{g}^{\top} \boldsymbol{H} \boldsymbol{g}$
      • There are three terms here: the original value of the function, the expected improvement due to the slope of the function, and the correction we must apply to account for the curvature of the function.
      • When $\boldsymbol{g}^{\top} \boldsymbol{H} \boldsymbol{g}$ is positive, the optimal stepsoze yields
        • $\epsilon^{*}=\frac{\boldsymbol{g}^{\top} \boldsymbol{g}}{\boldsymbol{g}^{\top} \boldsymbol{H} \boldsymbol{g}}$
        • worst case, optimal step is given by $\frac{1}{\lambda_{\max }}$,determined by the eigenvalues of the Hessian
    • In multiple dimensions, the condition number of the Hessian at this point measures how much the second derivatives differ from each other.
      • When the Hessian has a poor condition number, gradient descent performs poorly.
      • Poor condition number also makes choosing a good step size difficult.
  • Newton’s method, using information from the Hessian matrix to guide the search
    • is based on using a second-order Taylor series expansion to approximate $f(\boldsymbol{x}) $ near some point $\boldsymbol{x}^{(0)}$ :
      • $f(\boldsymbol{x}) \approx f\left(\boldsymbol{x}^{(0)}\right)+\left(\boldsymbol{x}-\boldsymbol{x}^{(0)}\right)^{\top} \nabla_{\boldsymbol{x}} f\left(\boldsymbol{x}^{(0)}\right)+\frac{1}{2}\left(\boldsymbol{x}-\boldsymbol{x}^{(0)}\right)^{\top} \boldsymbol{H}(f)\left(\boldsymbol{x}^{(0)}\right)\left(\boldsymbol{x}-\boldsymbol{x}^{(0)}\right)$
      • $\boldsymbol{x}^{*}=\boldsymbol{x}^{(0)}-\boldsymbol{H}(f)\left(\boldsymbol{x}^{(0)}\right)^{-1} \nabla_{\boldsymbol{x}} f\left(\boldsymbol{x}^{(0)}\right)$
      • When $f$ is a positive definite quadratic function, Newton’s method consists of applying equation once to jump to the minimum of the function directly.
      • Only for PD Hessian: Iteratively updating the approximation and jumping to the minimum of the approximation can reach the critical point much faster than gradient descent would.
        • This is a useful property near a local minimum, but it can be a harmful property near a saddle point, whereas gradient descent is not attracted to saddle points unless the gradient points toward them.
  • First-order optimization algorithms, optimization algorithms that use only the gradient, such as gradient descent
  • Second-order optimization algorithms, optimization algorithms that also use the Hessian matrix, such as Newton’s method
  • Lipschitz continuous
    • A Lipschitz continuous function is a function $f$ whose rate of change is bounded by a Lipschitz constant $\mathcal{L}$:
    • $\forall \boldsymbol{x}, \forall \boldsymbol{y},|f(\boldsymbol{x})-f(\boldsymbol{y})| \leq \mathcal{L}|\boldsymbol{x}-\boldsymbol{y}|_{2}$
    • quantify our assumption that a small change in the input made by an algorithm will have a small change in the output.

4.4 Constrained Optimization

  • constrained optimization: we may wish to find the maximal or minimal value of $f(\boldsymbol{x})$ for values of $\boldsymbol{x}$ in some set $\mathbb{S}$ .
  • Points $\boldsymbol{x}$ that lie within the set $\mathbb{S}$ are called feasible points 可行点 in constrained optimization terminology.
  • When possible, this method can be made more efficient by projecting the gradient into the tangent space of the feasible region before taking the step or beginning the line search.
  • The Karush–Kuhn–Tucker (KKT) approach provides a very general solution to constrained optimization.
    • we introduce a new function called the generalized Lagrangian or generalized Lagrange function.
      • $L(\boldsymbol{x}, \boldsymbol{\lambda}, \boldsymbol{\alpha})=f(\boldsymbol{x})+\sum_{i} \lambda_{i} g^{(i)}(\boldsymbol{x})+\sum_{j} \alpha_{j} h^{(j)}(\boldsymbol{x})$
    • and $\mathbb{S}=\lbrace\boldsymbol{x} | \forall i, g^{(i)}(\boldsymbol{x})=0 \text { and } \forall j, h^{(j)}(\boldsymbol{x}) \leq 0\rbrace$ with equality constraints $g^{(j)}$ and inequality constraints $h^{(j)}$
    • then
      • $\min _{\boldsymbol{x}} \max _{\boldsymbol{\lambda}} \max _{\boldsymbol{\alpha}, \boldsymbol{\alpha} \geq 0} L(\boldsymbol{x}, \boldsymbol{\lambda}, \boldsymbol{\alpha})$
    • has the same optimal objective function value and set of optimal points $\boldsymbol{x}$ as
      • $\min _{\boldsymbol{x} \in \mathbb{S}} f(\boldsymbol{x})$
    • This follows because any time the constraints are satisfied,
      • $\max _{\lambda} \max _{\boldsymbol{\alpha}, \boldsymbol{\alpha} \geq 0} L(\boldsymbol{x}, \boldsymbol{\lambda}, \boldsymbol{\alpha})=f(\boldsymbol{x})$
    • while any time a constraint is violated,
      • $\max _{\boldsymbol{\lambda}} \max _{\boldsymbol{\alpha}, \boldsymbol{\alpha} \geq 0} L(\boldsymbol{x}, \boldsymbol{\lambda}, \boldsymbol{\alpha})=\infty$
    • These properties guarantee that no infeasible point can be optimal, and that the optimum within the feasible points is unchanged.
  • KKT conditions
    • The gradient of the generalized Lagrangian is zero.
    • All constraints on both $\boldsymbol{x}$ and the KKT multipliers are satisfied.
    • The inequality constraints exhibit “complementary slackness”: $\boldsymbol{\alpha} \odot \boldsymbol{h}(\boldsymbol{x})=\mathbf{0}$.

4.5 Example: Linear Least Squares

  • $f(\boldsymbol{x})=\frac{1}{2}|\boldsymbol{A} \boldsymbol{x}-\boldsymbol{b}|_{2}^{2}$
    • Gradient descent
    • Newton’s method
  • suppose subject to the constraint $\boldsymbol{x}^{\top} \boldsymbol{x} \leq 1$
    • we introduce the Lagrangian
      • $L(\boldsymbol{x}, \lambda)=f(\boldsymbol{x})+\lambda\left(\boldsymbol{x}^{\top} \boldsymbol{x}-1\right)$
      • $\min _{\boldsymbol{x}} \max _{\lambda, \lambda \geq 0} L(\boldsymbol{x}, \lambda)$
        • We can find this value by performing gradient ascent on $\lambda$. To do so, observe
        • $\frac{\partial}{\partial \lambda} L(\boldsymbol{x}, \lambda)=\boldsymbol{x}^{\top} \boldsymbol{x}-1$
