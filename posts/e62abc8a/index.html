树莓派-Raspbian连接蓝牙设备

xlindo

2015-04-17

测试环境

Raspberry B+ Raspbian 某USB蓝牙模块 thinkpad 小红点 蓝牙鼠标 其余正常配件

内容

安装必要的组件

apt-get install bluetooth bluez-utils bluez-compat blueman

该语句可能会要求权限，可执行

#sudo apt-get install bluetooth bluez-utils bluez-compat blueman

安装完后，插上USB蓝牙适配器，运行

#lsusb

看有没有正确识别Bluetooth Dongle设备,例如:

Bus 001 Device 005: ID 0a12:0001 Cambridge Silicon Radio, Ltd Bluetooth Dongle (HCI mode)

然后运行

# service bluetooth status

看看Bluetooth服务是否正常运行，正常的话应该是

[ ok ] bluetooth is running.

博主此时重启了一下，感觉RPi遇到问题重启，有可能就神奇的解决了

连接阶段

用这个命令查看周围可见设备的地址和名称：

# hcitool scan

显示

aa:bb:cc:dd:ee:ff Thinkpad Bluetooth Laser Mouse

运行 （后面换成你设备的地址）：

#bluez-simple-agent hci0 aa:bb:cc:dd:ee:ff

在这边输入一个密码（0000），回车。 如果失败则会出现Creating device failed: ……之类的。请重试。

务必确认蓝牙设备是在请求配对的状态。

将这个新设备加入到信任列表：

#bluez-test-device trusted aa:bb:cc:dd:ee:ff yes

用这个命令手工开始连接：

#bluez-test-input connect aa:bb:cc:dd:ee:ff

有时可能蓝牙鼠标会自己断开，执行上面这条命令重连一下就OK了

大功告成!
